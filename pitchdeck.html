The Challenge

Five working days per case

Today, rescheduling a housing loan arrears application is a manual, paper-intensive process that takes an officer five working days per case.

For the beneficiary

  • Uncertain wait times for a decision
  • No visibility into the process
  • Multiple visits to service centres
  • Inconsistent outcomes across officers

For the Programme

  • High per-case processing cost
  • Officer time spent on repetitive analysis
  • Inconsistent rule application
  • Growing arrears caseload with fixed staff
The mandate: compress 5 days to near-instant →
Our Solution

Agent Sanad — governed, instant, auditable

A specialised AI casework agent that does the officer’s entire job — retrieve, validate, analyse, decide, explain — in under one second. The LLM reads and explains; deterministic code decides the money; a human officer owns exceptions.

Policy Compliance

Three official rules, enforced in code

The policy engine implements MOEI’s governance rules as deterministic gates — no LLM can override them, no officer can bypass them.

R1
20% Deduction Cap — Monthly deduction must not exceed 20% of verified income. Hard-checked. Surfaces as Pass/Fail chip on every report.
R2
Period Compliance — New schedule must not exceed original approved loan term. Per-path check fires TEN-01 if breached.
R3
Active Request Gate — Existing active application auto-rejects at gate one, before any computation. ACTIVE-01 fires.

Risk Assessment Matrix

  • INC-01 — Income contradiction between certificate and verification
  • OBL-01 — Financial obligations exceed 60% of income
  • FAM-01 — Low income per family member (confidence lowered)
  • HARD-01/02 — Hardship detection with verified/unverified branches
  • RSK-01 — Document injection attempts logged, policy unchanged
  • CAP-01/02 — Headroom analysis under the 20% cap
14 rule IDs total · Each with deterministic effect

Engineering Depth

Built to the IBM Research 7-skills playbook

Agent Sanad is engineered across all seven disciplines that separate a production agent from a demo experiment.

1. System Design

Explicit 8-state machine. Hard LLM/deterministic/human boundary. Not a free-form chatbot.

2. Tool & Contract Design

Pydantic v2 with extra="forbid" on every payload. Five typed adapter contracts.

3. Retrieval Engineering

Five fixture-backed adapters. Cached extraction fallback. No RAG ambiguity.

4. Reliability Engineering

Offline-first. Cached fallbacks. Stale-port guard. Auto graph-fallback. 165 tests.

5. Security & Safety

Untrusted documents (RSK-01). Read-only LLM. XSS-escaped UI. PII redaction interlock.

6. Evaluation & Observability

94.6% historical benchmark. Append-only audit. LangSmith-ready tracing (off by default).

7. Product Thinking

Dual surfaces: beneficiary (status only) and officer (full evidence). Evidence Repair Loop converts dead-end states into guided next steps. Exception Studio for case queue filtering.

Beyond the MVP

Production-ready capabilities

Optional but implemented: industry-standard tooling that proves the architecture scales to production.

Live Policy Configuration

MOEI can adjust thresholds (20% cap, headroom, obligations ratio) via PUT /admin/policy. No restart. No developer. Changes apply to the next decision.

What-If Simulator

“What if this beneficiary earned AED 5,000 instead of AED 16,000?” Post any override to /simulate/what-if/{case} and see the exact decision delta instantly.

Real-Time Streaming

SSE endpoint /demo/stream/{case} streams each step of the decision pipeline live — showing the agent “thinking” in real-time for demos and audit.

Batch Analysis & Docker

Run all 13 demo cases in one API call for comparison matrix. One-command deploy: docker-compose up with healthcheck, persistence, and restart policy.
